1. PRÉSENTATION DU PROJET
================================================================================

GME — Global Map Experience est une plateforme d'analyse géospatiale et de
reporting commercial à destination des entreprises. Elle permet de :

  - Analyser des zones géographiques (zones de chalandise, implantations,
    flux migratoires, sectorisation, etc.)
  - Créer des rapports visuels avec des indicateurs, graphiques et tableaux
    comparatifs
  - Visualiser des données sur des cartes interactives (MapLibre GL + Turf.js)
  - Importer, éditer et gérer des jeux de données (CSV, XLSX, connecteurs)
  - Exporter les résultats en PDF, XLSX, CSV, PNG
  - Gérer les accès utilisateurs, quotas, abonnements (Stripe), licences clients

================================================================================
3. STACK TECHNIQUE
================================================================================

--- FRONTEND (gme-front-master) ---

  - Vue.js 2        : Framework JavaScript (avec vue-class-component / vue-property-decorator)
  - TypeScript      : Typage statique
  - D3.js           : Rendu SVG de tous les graphiques
  - MapLibre GL     : Cartes interactives
  - Turf.js         : Opérations géospatiales (calculs de zones, distances...)
  - Vuex            : Gestion d'état centralisée
  - html2canvas     : Export PNG / PDF (capture du DOM)
  - canvg           : Conversion SVG pour export
  - ExcelJS         : Export XLSX
  - Webpack         : Bundler avec hot-reload

--- BACKEND (gme-server-app-master) ---

  - Node.js 16+     : Runtime JavaScript côté serveur
  - Hapi.js         : Framework HTTP
  - TypeORM         : ORM pour PostgreSQL
  - PostgreSQL       : Base de données relationnelle
  - Redis           : Cache
  - Cassandra       : Données séries temporelles (footfall)
  - Système de migrations SQL

6. SYSTÈME DE VISUALISATION — ARCHITECTURE DÉTAILLÉE
================================================================================

Le système suit un pattern Factory + Composition de Mixins :

  ReportTemplate (BDD)
      └── typeViz: string  (ex: "vbar", "multi-line", ...)
             │
             ▼
  report-indicator-graph.tsx
      └── graphForType(typeViz)   ←   factory.ts
             │
             ▼
  Composant Vue + D3 (ex: MultiLineChart)
      ├── VizMixin              (rendu D3, resize, interactions)
      ├── VizSingleTupleMixin   (données mono-territoire)
      │   OU
      │   VizMultiTuplesMixin   (données multi-territoires)
      └── CssVarsMixin          (couleurs via CSS variables)


--- TYPES DE VISUALISATION ACTUELLEMENT DISPONIBLES ---

  TypeViz         | Composant              | Usage typique
  ----------------|------------------------|-------------------------------
  vbar            | VBarChart              | Comparaison de valeurs simples
  hbar            | HBarChart              | Classements horizontaux
  multi-vbar      | MultiVBarChart         | Comparaison multi-territoires
  multi-hbar      | MultiHBarChart         | Classements multi-séries
  multi-line      | MultiLineChart         | Évolution temporelle
  multi-radar     | MultiRadarChart        | Profils comparatifs
  waterfall       | WaterfallChart         | Décomposition de variation
  radial          | RadialChart            | Répartition (donut/cercle)
  table           | TableChart             | Données tabulaires
  var-table       | VariableTableChart     | Tableau avec variables
  text            | TextChart              | KPI / valeur unique
  comment         | (non rendu)            | Annotations textuelles
  multi-table     | (non implémenté)       | Déclaré mais sans composant

7. COMMENT AJOUTER UN NOUVEAU TYPE DE GRAPHIQUE
================================================================================

Pour ajouter une nouvelle forme de représentation (ex. : camembert, scatter
plot, heatmap, bubble chart...), voici les 4 étapes et fichiers à modifier :

--- ÉTAPE 1 : Créer le composant graphique ---

  Fichier à créer :
    gme-front-master/src/components/charts/mon-nouveau-chart.tsx

  Structure à respecter :
    - Étendre VizMixin
    - Utiliser VizSingleTupleMixin (données simples)
      OU VizMultiTuplesMixin (données multi-séries)
    - Implémenter le rendu D3.js dans la méthode render() ou mounted()
    - Utiliser ResizableMixin pour l'adaptation à la taille

  Modèle recommandé : s'inspirer de radial.tsx (simple) ou
                      multi-line-chart.tsx (multi-séries)


--- ÉTAPE 2 : Enregistrer dans la factory ---

  Fichier à modifier :
    gme-front-master/src/components/charts/factory.ts

  Modifications :
    1. Ajouter une valeur dans l'enum TypeViz :
       ex : MON_GRAPHIQUE = 'mon-graphique',

    2. Ajouter un case dans la fonction graphForType() :
       ex : case TypeViz.MON_GRAPHIQUE: return MonNouveauChart;


--- ÉTAPE 3 : Exposer l'option dans l'interface d'édition ---

  Répertoire à modifier :
    gme-front-master/src/modules/analysis/engines/reporting/components/

  Objectif : ajouter le nouveau typeViz dans la liste déroulante de sélection
             du type de graphique lors de la création/édition d'un rapport.


--- ÉTAPE 4 : Côté backend (aucune migration nécessaire) ---

  Le champ typeViz dans report-template.ts est de type string libre.
  Aucune migration de base de données n'est nécessaire — la nouvelle
  valeur sera sauvegardée automatiquement.

9. CONTRÔLE D'ACCÈS AUX RAPPORTS
================================================================================

Les permissions sont gérées par des scopes hiérarchiques :

  Scope                                   | Accès accordé
  ----------------------------------------|--------------------------------
  analysis:reporting:enabled              | Accès en lecture aux rapports
  analysis:reporting:create               | Création de nouveaux rapports
  analysis:reporting:edit-self            | Modification de ses propres rapports
  analysis:reporting:edit-client          | Modification au niveau client
  analysis:reporting:edit-all-clients     | Administration globale
  analysis:reporting:share-internal-*     | Partage interne (collègues)
  analysis:reporting:share-external-*     | Partage externe (clients)

Ces scopes s'appliquent aux moteurs : reporting, implanting, prdt,
targeting, zone-chalandise.
